# OpenRoboOLP - 打包说明

## 快速开始

### 方法一：使用自动打包脚本（推荐）

```powershell
# 双击运行或在 PowerShell 中执行
.\build_win64.ps1
```

### 方法二：手动打包

```powershell
# 1. 激活虚拟环境（如果使用）
.venv\Scripts\Activate.ps1

# 2. 安装 PyInstaller
pip install pyinstaller pillow

# 3. 执行打包
pyinstaller --clean olp.spec
```

## 打包配置

### olp.spec 配置说明

```python
# 主程序入口
a = Analysis(['olp.py'])

# 需要打包的数据文件
datas = [
    ('logo.png', '.'),              # 软件图标
    ('models', 'models'),           # 机器人模型
    ('examples', 'examples'),       # 示例文件
    ('README.md', '.'),             # 说明文档
]

# 隐藏导入（避免模块未找到错误）
hiddenimports = [
    'PySide6',
    'PySide6.QtCore',
    'PySide6.QtGui',
    'PySide6.QtWidgets',
    'PySide6.QtOpenGLWidgets',
    'OpenGL',
    'OpenGL.GL',
    'numpy',
]

# 排除不需要的模块
excludes = [
    'tkinter',
    'matplotlib',
    'PIL',
    'scipy',
]

# 生成单文件可执行程序
exe = EXE(
    ...,
    console=False,  # 关闭控制台窗口
    icon='logo.png',  # 应用图标
)
```

## 常见问题

### Q: 打包后运行提示缺少模块？

A: 在 `hiddenimports` 列表中添加缺失的模块名称，重新打包。

### Q: 打包文件太大？

A: 可以在 `excludes` 列表中添加不需要的模块，减少文件大小。

### Q: 如何包含额外的数据文件？

A: 在 `datas` 列表中添加，格式为 `('源路径', '目标路径')`。

### Q: 打包过程中提示图标格式错误？

A: 确保已安装 Pillow 库：`pip install pillow`，或者将图标转换为 .ico 格式。

## 重新打包步骤

```powershell
# 1. 清理旧的构建文件
Remove-Item -Recurse -Force build, dist

# 2. 重新生成 spec 文件（可选）
pyinstaller olp.py

# 3. 编辑 olp.spec 配置（根据需要）

# 4. 使用 spec 文件打包
pyinstaller --clean olp.spec
```

## 发布说明

打包完成后，`dist/OpenRoboOLP.exe` 即为最终发布文件，可以直接分发给用户使用，无需安装 Python 环境。

**建议发布内容**：
- `OpenRoboOLP.exe` - 主程序
- `使用说明.md` - 使用文档
- `models/` - 示例机器人模型（可选）
- `examples/` - 示例文件（可选）
